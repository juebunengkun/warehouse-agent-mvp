# 指标口径库

## 销售额

- 业务含义：支付成功订单的实付金额。
- 计算口径：`SUM(pay_amount)`。
- 过滤条件：订单状态为 `PAID` 或 `FINISHED`，排除无效订单。
- 建议字段名：`sales_amount`。

## 订单数

- 业务含义：支付成功的去重订单数量。
- 计算口径：`COUNT(DISTINCT order_id)`。
- 过滤条件：订单状态为 `PAID` 或 `FINISHED`。
- 建议字段名：`order_count`。

## 支付用户数

- 业务含义：发生支付成功行为的去重用户数量。
- 计算口径：`COUNT(DISTINCT user_id)`。
- 过滤条件：订单状态为 `PAID` 或 `FINISHED`。
- 建议字段名：`pay_user_count`。

## 客单价

- 业务含义：平均每笔支付订单贡献的实付金额。
- 计算口径：`实付金额 / 支付订单数`。
- 除零规则：支付订单数为 0 时返回 0。
- 建议字段名：`avg_order_amount`。

## 转化率

- 业务含义：访问用户中最终完成支付的比例。
- 计算口径：`支付用户数 / 访问用户数`。
- 注意事项：需要同时具备访问明细和支付明细。
- 建议字段名：`conversion_rate`。

## 曝光UV

- 业务含义：报表统计粒度下发生商品或活动曝光的去重用户数。
- 计算口径：`COUNT(DISTINCT exposure_user_id)`。
- 过滤条件：只统计有效曝光事件，排除测试流量和风控黑名单用户。
- 建议字段名：`exposure_uv`。

## 点击UV

- 业务含义：报表统计粒度下发生点击行为的去重用户数。
- 计算口径：`COUNT(DISTINCT click_user_id)`。
- 过滤条件：只统计有效点击事件，排除测试流量和风控黑名单用户。
- 建议字段名：`click_uv`。

## 下单用户数

- 业务含义：报表统计粒度下创建有效订单的去重用户数。
- 计算口径：`COUNT(DISTINCT order_user_id)`。
- 过滤条件：订单为有效订单，不要求支付成功。
- 建议字段名：`order_user_count`。

## GMV

- 业务含义：有效订单创建时的商品成交总额，包含支付前取消订单。
- 计算口径：`SUM(gmv_amount)`。
- 过滤条件：排除测试订单、风控订单和无效订单。
- 建议字段名：`gmv_amount`。

## 实付金额

- 业务含义：支付成功订单的用户实际支付金额。
- 计算口径：`SUM(pay_amount)`。
- 过滤条件：订单状态为 `PAID` 或 `FINISHED`。
- 建议字段名：`pay_amount`。

## 优惠金额

- 业务含义：支付成功订单使用的优惠券、满减、补贴等优惠金额。
- 计算口径：`SUM(discount_amount)`。
- 过滤条件：订单状态为 `PAID` 或 `FINISHED`。
- 建议字段名：`discount_amount`。

## 退款金额

- 业务含义：退款发生日口径的退款金额，不回刷支付日期。
- 计算口径：`SUM(refund_amount)`。
- 过滤条件：退款单状态为退款成功，排除测试退款单。
- 建议字段名：`refund_amount`。

## 支付订单数

- 业务含义：支付成功的去重订单数。
- 计算口径：`COUNT(DISTINCT order_id)`。
- 过滤条件：订单状态为 `PAID` 或 `FINISHED`。
- 建议字段名：`pay_order_count`。

## 支付转化率

- 业务含义：点击用户中最终完成支付的比例。
- 计算口径：`支付用户数 / 点击UV`。
- 除零规则：点击UV为 0 时返回 0。
- 建议字段名：`pay_conversion_rate`。

## ARPU

- 业务含义：报表统计粒度下每个点击用户带来的平均实付金额。
- 计算口径：`实付金额 / 点击UV`。
- 除零规则：点击UV为 0 时返回 0。
- 建议字段名：`arpu`。
